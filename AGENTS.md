## 言語設定

**重要**: このプロジェクトでの作業は、原則として **常に日本語** を使用してください。

### 基本方針

1. **すべてのコミュニケーションは日本語で行う**
   - コード説明
   - コメント
   - ドキュメント
   - レビューコメント
   - 提案内容

2. **日本語でのコード記述**
   - コメントは日本語で記述
   - 変数名・関数名は適切な英語を使用（業界標準に従う）
   - ドキュメンテーション文字列は日本語で記述

3. **日本語での対応**
   - 質問への回答は日本語で行う
   - エラーメッセージの説明は日本語で行う
   - 提案や改善点は日本語で説明する

### 英語のまま使用してよいもの

以下は日本語化せず、そのまま英語で記述してください:

- 変数名・関数名・クラス名・ファイル名・ブランチ名
- 技術用語（API、HTTP、JSON、TypeScript など）
- 外部ライブラリ名・npm パッケージ名・コマンド名
- URL、ファイルパス、コードブロック内のコード
- コミットメッセージのプレフィックス（`feat:`, `fix:`, `docs:` など）
- プログラミング言語のキーワード、標準ライブラリの関数名

### コード例

```python
# 良い例：コメントは日本語
def calculate_total(items):
    """
    商品リストの合計金額を計算する

    Args:
        items: 商品のリスト

    Returns:
        合計金額
    """
    total = 0
    for item in items:
        total += item.price
    return total
```

## プルリクエスト (PR) は日本語で記載すること

AI エージェントが作成するすべての PR は、**タイトル・本文ともに日本語で記載してください**。これは例外なく適用されます。

### 必須事項

- PR タイトルは日本語で記述する
- PR 本文（Summary / Test plan などのセクションを含む）は日本語で記述する
- セクション見出し（例: `## 概要`, `## 変更点`, `## テスト計画`）も日本語にする
- レビューコメントへの返信も日本語で行う

### PR テンプレート

本リポジトリには `.github/PULL_REQUEST_TEMPLATE.md` が用意されています。PR 作成時はそのテンプレートに沿って日本語で記述してください。

簡易的なフォーマット例:

```markdown
## 概要
<このPRで何を、なぜ変更したかを1〜3個の箇条書きで>

## 変更点
- <主要な変更点>

## テスト計画
- [ ] <検証項目>
```

## コミットメッセージ

コミットメッセージの本文も日本語で記述してください。プレフィックスは英語のまま使用します。

例:
```
feat: ユーザー一覧画面を追加

- ユーザー検索機能を実装
- ページネーション機能を追加
```

## ドキュメント作成

- README やその他のドキュメントは日本語で記述
- 技術仕様書は日本語で記述
- ユーザーマニュアルは日本語で記述

## ブランチ運用ポリシー

**重要**: `main` ブランチへの直接的なコミットおよびプッシュは **絶対に禁止** です。

### 基本ルール

1. **`main` ブランチでの作業禁止**
   - `main` ブランチに直接コミットしてはいけません
   - `main` ブランチに直接プッシュしてはいけません

2. **作業ブランチの作成**
   - コミットやプッシュの指示を受けた場合、まず現在のブランチを確認してください
   - 現在 `main` ブランチにいる場合は、以下の手順を実行してください:
     1. ユーザーに対して、`main` ブランチでの作業は禁止されていることを説明する
     2. 適切な名前の新しいブランチを作成することを提案する
     3. ユーザーの同意を得てから新しいブランチを作成する
     4. 新しいブランチに切り替えてから作業を開始する

3. **ブランチ名の命名規則**
   - `feature/` — 新機能の追加
   - `fix/` — バグ修正
   - `docs/` — ドキュメントのみの変更
   - `refactor/` — リファクタリング
   - `test/` — テストの追加や修正

### 作業フロー例

```bash
# 現在のブランチを確認
git branch

# main ブランチにいる場合は新しいブランチを作成
git checkout -b feature/new-feature

# 作業を実施
# ...

# コミットとプッシュ（新しいブランチで実施）
git add .
git commit -m "feat: 作業内容の説明"
git push origin feature/new-feature
```

### 例外事項

- この制限に例外はありません
- 緊急の場合でも、必ずブランチを作成してから作業してください

## やってはいけないこと（要約）

- `main` への直接コミット／直 push、`main` への force push
- ユーザーの明示的な許可なしの依存関係の追加・破壊的変更
- 機密情報（`.env`、認証情報）のコミット
- ユーザーの明示的な許可なしに `git config` を変更したり、フックをスキップ（`--no-verify` など）したりすること
